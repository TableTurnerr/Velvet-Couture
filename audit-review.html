Lean into

Don't push

  • "kabob" / "kebab" / "shawarma" / "mandi" / "masgoof" — 2 mentions total. Either bring the dishes back to the menu (audit-aligned strategy) or stay the course on bakery-café and remove the few stragglers, e.g. "kabob restaurant Allen TX" metaTitle / popularDish entries that don't reflect reality.

  • "Mediterranean" — 92 mentions is enough. Avoid over-using it; it dilutes the Iraqi-specific positioning the audit identified as the moat.
  • Generic adjectives ("authentic", "traditional", "delicious") — already heavy; further repetition won't move rankings.

What this tells us, in three lines

  • The Iraqi-specific long tail is working. Top-3 organic for iraqi bakery dallas, iraqi bakery plano, iraqi breakfast richardson, iraqi breakfast dallas, iraqi sweets dallas. #1 for iraqi bakery richardson. That's the moat the audit predicted, now measurable.
  • The legacy domain is eating our authority. Two old domains are still resolving, still ranking, and still pulling click share from the canonical site on branded and baklava queries. Until both are 301'd, every backlink and every Google "this is the same business" signal stays split.
  • Dessert + competition-heavy queries aren't rankable from copy alone. Unranked for kunafa dallas, kunafa richardson, baklava dallas, fatayer richardson, ladyfingers znood al sit dallas, halal catering richardson, and every "best baklava in [city]" query. These need a combination of backlinks, more reviews, video/image freshness, or all three — pure on-page content has done what it can.

Capture method & caveats

  • Tool: Playwright headless Chromium against google.com/search?q=...&gl=us&hl=en&num=20&pws=0.
  • Date: 2026-05-21.
  • Single capture per query. Google rankings fluctuate within a day; a 5-run median would be more reliable.

Recommended next moves

  • Strategic

    Resolve the bakery-café vs. restaurant identity question

    The audit was written for a kabob/shawarma/mandi restaurant. The site sells a bakery + breakfast café. Both are valid, but the site still carries traces of the old positioning (the slug kabob restaurant allen tx in neighborhoods.ts keywords, "Salam Grill" in alternateName). Confirm with the owner which business this actually is in 2026, then either (a) add a kabob/grill page and surface mixed-grill content, or (b) sweep the codebase and remove kabob/mandi residue so the bakery-café story is clean end-to-end.

  • Phase 3

    Move ordering on-domain (or a strong "order direct" framing)

    This was the audit's top recommendation (#3.3) and is still open. ZingMyOrder commission is opaque, but ~25% is the industry default. Either embed an on-domain iframe with the existing ZingMyOrder cart, migrate to Toast / Square Online ordering with a domain-native checkout, or at minimum add a "save 25% — order direct" banner above the order button so users understand why direct beats DoorDash.

  • Quick

    Run an actual Lighthouse capture and attach the numbers

    The audit estimated LCP 3.5–4.5s, CLS 0.15+, weight 2–3MB. The current build should beat all three but the repo has no proof. Run npm run build, npm run serve-out, then Lighthouse against http://localhost:3000/, /menu/, /bakery/, /catering/, and one neighborhood page. Drop the numbers into a perf.md at repo root. Verifies the §5 promise.

  • Phase 3

    Replace mailto catering form with a real endpoint

    CateringForm currently builds a mailto: link. Cart abandonment risk is high — many users don't have a mail client configured on desktop, and on mobile the handoff is jarring. Wire it to a Cloudflare Worker endpoint or a Formspree/Web3Forms POST that emails the team directly and logs leads in a sheet. Static export friendly, ~1 hour of work, measurable on catering inquiry KPI.

  • Quick

    Add a live Google Review widget

    TrustBar shows a hardcoded "1,899 reviews · 4.4★" pulled from RESTAURANT.reviewCount. Replacing that with a live Google Places API badge (or a static badge regenerated nightly via a small fetch script) puts the social-proof number on a freshness loop. The audit treated 1,892 as the moat — proving it's still growing live is more persuasive than a number in a TypeScript file.

  • Strategic

    Open TikTok + YouTube channels and embed

    The audit's §10 called these out specifically: "sticky cheese kunafa" videos cross 1M views routinely, samoon and tandoor process content is defensible, kanafa pulls are dwell-time gold. Currently zero presence. Even six TikTok shorts and a YouTube playlist would unlock the "iraqi dessert near me" / "sticky cheese kunafa dallas" trend keywords and create a content moat that no Richardson competitor has.

  • Quick

    Move from albaghdady.bakery@gmail.com to info@al-baghdady.com

    The typo from the audit (nfo@salamgrill.com) is fixed, but the replacement is a personal Gmail. Google Workspace at $7/mo gives a branded address, signals trust on schema/contact, and lets the catering form's reply-to look professional. One DNS MX record + 30 minutes.

  • Quick

    Trim long specialty meta titles to ≤60 chars

    Some specialty meta titles run 72–78 characters ("Lady Fingers in Richardson, TX | Al-Baghdady — Iraqi Bakery & Café" = 71). Google truncates around 60. Drop "Iraqi Bakery & Café" from the trailing brand string on the longer slugs — the H1 + breadcrumb still carries that context.

  • Build

    Ship the planned /[dish]-in-[city]/ matrix route

    dishes.ts + matrix.ts are already in place — the infrastructure is half-built. With 10 cities × ~6 hero dishes gated by MATRIX_ALLOWLIST, that's 30–60 new long-tail pages targeting "baklava in plano", "kunafa in frisco" style queries. Each page is a 200-word programmatic template; the SEO upside is significant and the marginal cost per page is near zero.

  • Quick

    Add a Press / Coverage page

    PressStrip carries the D Magazine quote on home, but the audit (§9 + §18) referenced Dallas Observer "Dallas' Best Iraqi Restaurants", Dallas OCA "Iraqi Restaurant Dallas", Restaurant Guru. Consolidate those into /press/ with each outlet's headline, date, and a link. Strong trust signal, low effort, helps "best iraqi restaurant dallas" rank via citation density.

  • Highest-leverage open item

    301 the two legacy domains. They're still live and outranking the new site.

    Confirmed via direct fetch on 2026-05-21: https://albaghdadyrestaurant.com/ returns 200 OK with the old ZingMyOrder page, title "Al Baghdady Restaurant…Burgers, Mediterranean Restaurant". https://albaghdadybakeryandcafe.com/ also returns 200. In live SERPs, the legacy site is currently #1 for "albaghdady" (the new site is #2), #3 for "baklava richardson" (the new site is #7), #7 for "best middle eastern food in richardson", and #9 for "best middle eastern in richardson tx". Every visit to those pages is a visit that doesn't accrue authority to al-baghdady.com. Configure both registrars to point at a host that returns HTTP 301 Location: https://al-baghdady.com/<path> for every URL.
